Missing Content Opportunities

CONTENT GAP

No Dedicated Silver Purity Chart / Reference Page

Users searching "silver purity chart", "silver grades explained", or "925 vs 999 silver" have no single reference page. The hallmarks guide exists but a comprehensive purity comparison page with alloy compositions, use cases, and value table would rank well for informational queries and serve as a strong internal linking hub.

CONTENT GAP

silver-news-today.html is One Monolithic File

At 14,666 words, the news page is a single file containing all news items. This means: (1) only one URL can rank for any news query, (2) older news pushes down newer content, (3) no individual article URLs to build links to. The architecture should be individual article pages (/silver-news/article-slug/) with a news index page.

LOW

No "Silver Price History" or "Silver Price Chart" Page

High-volume queries like "silver price history", "silver price 10 years", "silver price 2024 vs 2025" are not covered. An interactive historical price chart page would attract informational search traffic and support the "live price" hub.

Mobile Issues Found

MEDIUM

Email Subscribe Form May Overflow on 320px Screens

The homepage email capture form uses min-width:200px on the input and flex-wrap:wrap on the container. On 320px screens (iPhone SE 1st gen, some Android budget phones), the input + button may force an extra line break awkwardly, but should still function. Test on an actual 320px viewport.

Action: Change to width:100% on the input at @media(max-width:480px) and stack the button below it explicitly.
LOW

Hero H1 Font Size Could Cause Wrapping on Small Screens

The homepage H1 uses clamp(2.8rem, 4.8vw, 4.2rem). On a 375px iPhone, 4.8vw = 18px which clamps to 2.8rem = 44.8px — a very large headline. "Scrap Silver Calculator" at 44px may wrap to 3 lines on 375px, pushing the calculator below the fold.

Action: Test on 375px viewport. Consider reducing the clamp minimum to 2.2rem or adding a @media(max-width:400px) override.

HIGH

Email Subscribe Has No List Management — Subscribers Receive Nothing

The /api/subscribe endpoint collects an email address and sends a notification email to the site owner via Zoho SMTP. That's it. Subscribers never receive a welcome email, a confirmation, or any daily silver price updates. Anyone who subscribes is left in silence. This makes the "Get Daily Silver Prices in Your Inbox" promise false — which is a trust issue and could be flagged by email service providers if a complaint is raised.

Action (Priority): Integrate a proper email service (Mailchimp, ConvertKit, or Brevo — all have free tiers). Move the subscriber list there. Set up an automated welcome email and a daily price digest. The current Zoho-forward approach is not scalable and breaks the user promise.
MEDIUM

No Exit Intent or Second-Chance Capture

When a user finishes calculating and navigates away, there is no exit intent capture, no "bookmark this" prompt, and no re-engagement mechanism beyond the sticky mobile CTA. For a tool site with high bounce rate (users get their answer and leave), this is a missed opportunity to build a returning audience.

Action: Consider a lightweight exit-intent modal (triggers on cursor leaving viewport on desktop, or on back-button press on mobile) offering the email subscribe or a link to bookmark the page.
MEDIUM

No Retargeting Pixel Beyond GA4

GA4 is installed and configured. However, there is no Meta Pixel, LinkedIn Insight Tag, or Pinterest Tag for retargeting ads. If you plan to run paid acquisition, you have no audience data being built. The Pinterest domain verification (p:domain_verify) is present on the homepage — Pinterest Tag should be added to match.

Action: Add Pinterest Tag (since the domain is already verified). Consider Meta Pixel if Facebook/Instagram ads are in the plan.
